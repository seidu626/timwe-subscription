help_outline How Mobplus Postbacks Work

Postback Flow

  1. User subscribes via landing page with click_id and pub_id from Mobplus tracking URL
  2. Transaction created in acquisition-api with attribution data stored
  3. TIMWE processes subscription and eventually charges the user
  4. subscription-external receives charge-success callback from TIMWE
  5. subscription-external calls POST /internal/acquisition/charge-success
  6. acquisition-api marks transaction as CHARGED and queues conversion postback
  7. postback-dispatcher picks up the outbox entry and fires S2S postback to Mobplus

Postback URL Format

The rendered URL should look like:

http://m.mobplus.net/c/p/CAMPAIGN_KEY?txid=CLICK_ID&pub_id=PUB_ID

Status Meanings

  • PENDING - Waiting to be processed
  • PROCESSING - Currently being sent
  • SUCCESS - Successfully delivered (HTTP 2xx)
  • FAILED - Delivery failed, may retry
  • DLQ - Dead Letter Queue (max retries exceeded)
